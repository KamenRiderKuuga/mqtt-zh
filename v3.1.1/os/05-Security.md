# 5 安全

## 5.1 介绍

本章仅供参考，为**非规范性内容**。但是，强烈建议提供 `TLS` [RFC5246] 的服务器实现**应该**使用 TCP 端口 8883（IANA 服务名：secure-mqtt）。

解决方案提供者需要考虑许多威胁。例如：

- 设备可能被攻破
- 客户端和服务器上的静态数据可能被访问
- 协议行为可能产生副作用（例如"时序攻击"）
- 拒绝服务（DoS）攻击
- 通信可能被拦截、修改、重路由或泄露
- 注入伪造的控制报文

`MQTT` 解决方案通常部署在敌对的通信环境中。在这种情况下，实现通常需要提供以下机制：

- 用户和设备的认证
- 服务器资源访问的授权
- `MQTT Control Packets` 及其包含的应用数据的完整性
- `MQTT Control Packets` 及其包含的应用数据的隐私

作为一种传输协议，`MQTT` 只关注消息传输，实现者有责任提供适当的安全功能。这通常通过使用 `TLS` [RFC5246] 来实现。

除了技术安全问题外，还可能存在地理（例如美欧安全港协议 [USEUSAFEHARB]）、行业特定（例如 PCI DSS [PCIDSS]）和监管方面的考虑（例如萨班斯-奥克斯利法案 [SARBANES]）。

## 5.2 `MQTT` 解决方案：安全和认证

实现可能希望符合特定的行业安全标准，例如 NIST 网络安全框架 [NISTCSF]、PCI-DSS [PCIDSS]、FIPS-140-2 [FIPS1402] 和 NSA Suite B [NSAB]。

关于在 NIST 网络安全框架 [NISTCSF] 中使用 `MQTT` 的指南可以在 `MQTT` 补充出版物《`MQTT` 和 NIST 改善关键基础设施网络安全框架》[MQTT NIST] 中找到。使用经过行业验证、独立验证和认证的技术将有助于满足合规性要求。

## 5.3 轻量级加密和受限设备

高级加密标准 [AES] 和数据加密标准 [DES] 已被广泛采用。

ISO 29192 [ISO29192] 为专门针对受限"低端"设备性能优化的加密原语提出了建议。

## 5.4 实现说明

在实现或使用 `MQTT` 时，有许多安全问题需要考虑。以下部分不应被视为"检查清单"。

实现可能希望实现以下部分或全部内容：

### 5.4.1 服务器对客户端的认证

`CONNECT` 报文包含用户名和密码字段。实现可以选择如何使用这些字段的内容。它们可以提供自己的认证机制，使用外部认证系统如 `LDAP` [RFC4511] 或 `OAuth` [RFC6749] 令牌，或利用操作系统认证机制。

以明文形式传递认证数据、对此类数据进行混淆处理或不需要认证数据的实现应该意识到这可能导致中间人攻击和重放攻击。5.4.5 节介绍了确保数据隐私的方法。

客户端和服务器之间的虚拟专用网络（`VPN`）可以确保数据仅从授权客户端接收。

在使用 `TLS` [RFC5246] 时，服务器可以使用客户端发送的 `SSL` 证书来认证客户端。

实现可以允许通过应用消息从客户端向服务器发送凭证进行认证。

### 5.4.2 服务器对客户端的授权

实现可以根据客户端提供的信息（如用户名、`Client Identifier`、客户端的主机名/IP 地址）或认证机制的结果来限制对服务器资源的访问。

### 5.4.3 客户端对服务器的认证

`MQTT` 协议不是信任对称的：它没有提供客户端认证服务器的机制。

在使用 `TLS` [RFC5246] 时，客户端可以使用服务器发送的 `SSL` 证书来认证服务器。从单个 IP 地址为多个主机名提供 `MQTT` 服务的实现应该了解 RFC 6066 第 3 节中定义的 `TLS` 服务器名称指示扩展 [RFC6066]。这允许客户端告诉服务器它正在尝试连接的服务器主机名。

实现可以允许通过应用消息从服务器向客户端发送凭证进行认证。

客户端和服务器之间的 `VPN` 可以确保客户端连接到预期的服务器。

### 5.4.4 应用消息和控制报文的完整性

应用程序可以独立在其应用消息中包含哈希值。这可以在网络传输和静态存储时提供发布控制报文内容的完整性。

`TLS` [RFC5246] 提供哈希算法来验证网络传输数据的完整性。

使用 `VPN` 连接客户端和服务器可以在 `VPN` 覆盖的网络段上提供数据完整性。

### 5.4.5 应用消息和控制报文的隐私

`TLS` [RFC5246] 可以对网络传输的数据进行加密。存在一些有效的 `TLS` 密码套件包含 NULL 加密算法，该算法不对数据进行加密。为确保隐私，客户端和服务器应该避免使用这些密码套件。

应用程序可以独立加密其应用消息的内容。这可以在网络传输和静态存储时提供应用消息的隐私。但这不能为应用消息的其他属性（如 `Topic Name`）提供隐私。

客户端和服务器实现可以为静态存储的数据（如作为 `Session` 一部分存储的应用消息）提供加密存储。

使用 `VPN` 连接客户端和服务器可以在 `VPN` 覆盖的网络段上提供数据隐私。

### 5.4.6 消息传输的不可否认性

应用程序设计者可能需要考虑适当的策略来实现端到端的不可否认性。

### 5.4.7 检测客户端和服务器的攻破

使用 `TLS` [RFC5246] 的客户端和服务器实现应该提供能力，确保在发起 `TLS` [RFC5246] 连接时提供的任何 `SSL` 证书与正在连接的客户端或正在连接的服务器的主机名相关联。

使用 `TLS` [RFC5246] 的客户端和服务器实现可以选择提供检查证书撤销列表（CRLs [RFC5280]）和在线证书状态协议（OSCP [RFC6960]）的能力，以防止使用已撤销的证书。

物理部署可以将防篡改硬件与应用消息中传输的特定数据相结合。例如，仪表可能具有嵌入式 GPS 以确保它不会在未授权的位置使用。[IEEE 802.1AR] 是一个使用加密绑定标识符来实现设备身份认证机制的标准。

### 5.4.8 检测异常行为

服务器实现可以监控客户端行为以检测潜在的安全事件。例如：

- 重复的连接尝试
- 重复的认证尝试
- 连接的异常终止
- 主题扫描（尝试发送或订阅许多主题）
- 发送无法投递的消息（主题没有订阅者）
- 连接但不发送数据的客户端

服务器实现可以断开违反其安全规则的客户端。

检测到不良行为的服务器实现可以实现基于标识符（如 IP 地址或 `Client Identifier`）的动态阻止列表。

部署可以使用网络级控制（如果可用）来基于 IP 地址或其他信息实施速率限制或阻止。

### 5.4.9 其他安全考虑

如果客户端或服务器 `SSL` 证书丢失或被认为可能已泄露，应该撤销它们（使用 CRLs [RFC5280] 和/或 OSCP [RFC6960]）。

丢失或被认为已泄露的客户端或服务器认证凭证（如用户名和密码）应该被撤销和/或重新颁发。

对于长寿命连接：

- 使用 `TLS` [RFC5246] 的客户端和服务器实现应该允许会话重协商以建立新的加密参数（替换会话密钥、更改密码套件、更改认证凭证）。

- 服务器可以断开客户端并要求它们使用新凭证重新认证。

受限设备和受限网络上的客户端可以使用 `TLS` 会话恢复 [RFC5077]，以降低重新连接 `TLS` [RFC5246] 会话的成本。

连接到服务器的客户端与连接到同一服务器并被授权在同一 `Topic` 上发布数据的其他客户端之间存在传递信任关系。

### 5.4.10 SOCKS 的使用

客户端的实现应该意识到某些环境需要使用 `SOCKSv5` [RFC1928] 代理来进行出站网络连接。一些 `MQTT` 实现可以通过使用 `SOCKS` 来利用替代的安全隧道（例如 `SSH`）。在选择使用 `SOCKS` 时，实现应该同时支持匿名和用户名密码认证的 `SOCKS` 代理。在后一种情况下，实现应该意识到 `SOCKS` 认证可能以明文形式发生，因此应该避免使用相同的凭证连接到 `MQTT` 服务器。

### 5.4.11 安全配置文件

实现者和解决方案设计者可能希望将安全视为一组可以应用于 `MQTT` 协议的配置文件。下面展示了一个分层安全层次结构的示例。

#### 5.4.11.1 明文通信配置文件

使用明文通信配置文件时，`MQTT` 协议在没有额外安全通信机制的开放网络上运行。

#### 5.4.11.2 安全网络通信配置文件

使用安全网络通信配置文件时，`MQTT` 协议在具有安全控制（例如 `VPN` 或物理安全网络）的物理或虚拟网络上运行。

#### 5.4.11.3 安全传输配置文件

使用安全传输配置文件时，`MQTT` 协议在物理或虚拟网络上运行，并使用 `TLS` [RFC5246]，提供认证、完整性和隐私。

`TLS` [RFC5246] 客户端认证可以附加于或替代由用户名和密码字段提供的 `MQTT` 客户端认证。

#### 5.4.11.4 行业特定安全配置文件

预计 `MQTT` 协议将被设计到行业特定的应用配置文件中，每个配置文件定义一个威胁模型和用于应对这些威胁的特定安全机制。针对特定安全机制的建议通常来自现有工作，包括：

[NISTCSF] NIST 网络安全框架
[NIST7628] NISTIR 7628 智能电网网络安全指南
[FIPS1402] 加密模块安全要求（FIPS PUB 140-2）
[PCIDSS] PCI-DSS 支付卡行业数据安全标准
[NSAB] NSA Suite B 密码算法
